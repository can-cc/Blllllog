# Http缓存介绍

web 缓存是一个看上去简单，实际比较头疼的问题。

在 web 应用中应用缓存，一般情况上是在 header 上做文章， 通过 header 告诉浏览器应该如何缓存，其实用的都是浏览器的功能。

## 客户端缓存
客户端缓存值的是客户端（通常是浏览器）第一次请求完成后把一些网页资源都保存在本地，那么在下一次请求的时候，如果符合缓存条件，就直接把本地的资源拿出来用。w
